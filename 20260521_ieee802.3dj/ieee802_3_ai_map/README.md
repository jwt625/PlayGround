# IEEE 802.3 AI Relationship Map

Files:

- `ieee8023-ai-relationships.svg` - outsider-facing map with logical relationships, timeline, topics, officers, and visible contributor ecosystems.
- `ieee8023-ai-relationships.png` - rendered preview of the SVG.
- `participant_summary.json` - visible public affiliation counts used for per-node company labels.
- `timeline_data.json` - generated timeline ranges, lanes, source files, and precision notes.
- `high_level_pages/` - cached high-level IEEE 802.3 source pages used for the map.
- `high_level_pages.json` - manifest for the cached high-level pages.

## Design Intent

The map uses two simultaneous views:

1. Logical relationships: how E4AI, 802.3dj, 400GPL, 200GMMF/802.3ds, 802.3dq, COM, and adjacent workstreams relate technically.
2. Timeline: how the work evolved from B400G to 802.3df to 802.3dj, then toward 400G/lane, while the MMF path moved from 200GMMF to 802.3ds.

The timeline is generated by `scripts/generate_8023_timeline.py`. It computes first/last cached meeting dates or public-index dates, assigns lanes by workstream, and creates sublanes when concurrent blocks would collide.

The key-player strip separates:

- IEEE roles: officers from the IEEE 802.3 contacts page.
- Per-node company labels: top visible affiliation counts from cached presentation metadata and meeting-page affiliation columns.
- Contributor ecosystems: recurring public-deck affiliations and liaison/external-ecosystem inputs.

Important caveat: IEEE 802.3 participation is individual-based. Company names in the map should be read as visible contributor affiliations or ecosystem inputs, not formal owners or sponsors of a standards direction.
