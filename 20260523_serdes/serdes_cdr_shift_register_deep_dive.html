CDR / PLL And Shift Register Deep Dive

Behavioral mixed-signal view: UI = 8.93 ps, 16 UI word = 143 ps. New samples enter at right.

1. CDR Scope: Data, Edges, And Recovered Sampling Ticks

The recovered sample phase starts wrong, then the CDR nudges it using data transitions. This is a conceptual bang-bang CDR / PLL or phase-interpolator model, not transistor-level loop design.

2. CDR Loop Internals: Phase Detector, Charge Accumulation, Loop Filter, Phase Generator

UP/DOWN/HOLD decisions add or remove charge/control count. The loop filter smooths that control value, and the phase generator shifts the sampling phase.

3. Basket Register Bank: How A Sample Picks A Storage Slot

The modulo-16 counter enables exactly one storage basket per UI. The output latch captures all baskets together at the word boundary.

4. True Shift Chain: Serial-In / Parallel-Out Register

A different hardware-flavored view: every UI moves values through a chain. The final output latch still retimes the full word onto one common word clock.

5. Shift Mechanism Deep Dive: Master / Slave Latches And Snapshot Timing

A flip-flop stage is shown as two latches. The master samples while CLK is low, the slave publishes after the rising edge, and clock-to-Q delay prevents the new value from being consumed until the next edge.
