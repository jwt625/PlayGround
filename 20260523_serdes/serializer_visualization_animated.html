Animated Pure SerDes Visualization

Rolling oscilloscope-style view: new samples enter at right, older waveform history moves left. UI = 8.93 ps. Word period = 143 ps.

1. Animated Serializer Mux Tree / Trace Evolution

All mux-tree streams are shown. The visible window is three word periods wide; the right edge is the current sampling time.

2. Animated Final 2:1 Interleaving Zoom

The final one-word window shows even and odd streams feeding a 56 GHz controlled mux; the 112 Gb/s output advances one UI per bit.

3. Animated Latch / Mux Path Detail: Generating S1.0

This panel follows one first-stage example from the mux tree. In this ordering, S1.0 interleaves b0 and b8; each wire has a small rolling waveform, and active/high paths glow more strongly.

4. Latch Edge Regeneration Zoom

Conceptual model only: a slow input edge is sampled and relaunched as a sharper edge aligned to a clean clock edge.

5. Latch -> 2:1 Mux -> Latch Panel

Active sampling and relaunch stages re-buffer the mux output at each stage.

1. Animated Deserializer Demux Tree / Trace Evolution

The deserializer samples the 112 Gb/s serial stream, then routes alternating bits through staged 1:2 demuxes until 16 word-rate lanes are reconstructed.

2. Shift Register Word Assembly

A second equivalent view: each recovered UI sample fills one position in a 16-bit register; a divide-by-16 word-valid pulse captures the full parallel word.

3. Sampler / CDR / Selected Lane Detail

The CDR is shown conceptually as recovered sampling phase, not as a transistor-level PLL. The highlighted lane follows the currently sampled bit position.
